    KMP算法

    建于：2022-10-03 17:31:07 Monday 1143字 4分
    CC BY 4.0（除特别声明或转载）

    一些概念补充

    前缀：除最后一个字符外，字符串的所有头部子串；
    后缀：除第一个字符外，字符串的所有尾部子串；
    部分匹配值：字符串的前缀与后缀的最长相等前后缀长度。

    [!example] 案例
    字符串 ‘ababa’的前缀{a,ab.aba,abab},后缀{a,ba,aba,baba},$前缀\cap后缀$ = {a,aba}，公共元素有两个，最长相等前后缀长度为3，即部分匹配值为3。

    思路

    发生失配时，在主串中已经匹配的的子串，如果有相等的前缀与后缀，则可以直接将，当前对齐到前缀起始位置的模式串，对齐到后缀起始位置，这中间将减少匹配次数。

    实现

    根据模式串的信息计算部分匹配值，进一步得到next数组
    \(next[x] = \begin{cases} 0 ,x=1\\ 1,x=2\\ 部分匹配值[x-1]+1,x>2\\ \end{cases}\)

    在朴素模式匹配的基础上，利用next数组，实现匹配过程中主串指针i不回溯，发生是失配则j变为$next[j]$，以减少匹配次数，降低最坏时间复杂度到O(m+n)。

    int Index_KMP(SString S, SString T, int next[]) {
	int i = 1, j = 1;
	while (i <= S.length && j <= T.length) {
		if (j = 0 || S.ch[i] == T.ch[j]) {
			++i;
			++j;//继续比较后继字符
		}
		else
			j = next[j];//模式串向右移动
	}
	if (j > T.length)
		return i - T.length;//匹配成功
	else
		return 0;
}

    进一步优化

    上述的KMP算法中，使用到next数组来变更j表示的指向位序。在发生失配时，即$S[i] \neq T[j]$ ，j的值变更后，如果T[j]的值并未改变，则会再次发生失配，$S[i] \neq T[j]$依然成立，需要再次变更j的值。此处具有优化空间，可以将next数组中的部分有待优化的值，覆盖为迭代后的最终值，实现进一步减少匹配次数。

    本质上只是优化了next数组，将新的数组命名为nextval数组。

    int* generateNextval(SString T, int* next, int* nextval) {
	nextval[1] = 0;
	for (int j = 2; j <= T.length; j++) {
		if (T.ch[next[j]] == T.ch[j])
			nextval[j] = nextval[next[j]];
		else
			nextval[j] = next[j];
	}
	return nextval;
}
